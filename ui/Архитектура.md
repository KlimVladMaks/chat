## Архитектура модуля `ui` (Jetpack Compose)

Предлагаю максимально простой подход **без использования `NavHost`** — всё управление состоянием и навигацией происходит через `MainViewModel`. Это гарантирует:
- Сохранение открытого чата при повороте.
- Единый источник истины.
- Простую обработку кнопки «Назад».
- Отсутствие проблем с синхронизацией между портретом и ландшафтом.

### Общая структура пакетов `ui`

```
ui/
├── MainActivity.kt
├── components/          # Переиспользуемые компоненты (MessageItem, ChannelItem, LoadingIndicator и т.д.)
├── screens/
│   ├── ChannelsScreen.kt
│   ├── ChatScreen.kt
│   ├── FullImageScreen.kt
│   └── LoginScreen.kt
└── theme/               # Цвета, типографика, темы (если нужно)
```

### Ключевые принципы

- **Одна Activity** – `MainActivity`, внутри неё корневой `ChatApp`-композабл.
- **Ориентация** – определяем через `LocalConfiguration.current.orientation`.
- **Глобальное состояние** – `MainViewModel` (живёт на уровне Activity, не пересоздаётся при повороте).
- **Локальные состояния** – `ChannelsViewModel`, `ChatViewModel` (привязаны к `viewModel()` с ключами для сохранения при повороте).
- **Навигация** – на основе полей `MainState`:
  - `isAuthenticated` – показывать `LoginScreen` или основной контент.
  - `fullImagePath` – если не `null`, поверх всего показывать `FullImageScreen`.
  - В портрете: если `selectedChannel == null` → `ChannelsScreen`, иначе `ChatScreen`.
  - В ландшафте: всегда две панели; правая панель зависит от `selectedChannel`.

### Основные компоненты

#### 1. `MainActivity`

```kotlin
class MainActivity : ComponentActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        val repository = (application as ChatApplication).repository // или создаём через DI
        val mainViewModel = ViewModelProvider(
            this,
            MainViewModelFactory(repository)
        ).get(MainViewModel::class.java)

        setContent {
            ChatApp(mainViewModel, repository)
        }
    }
}
```

#### 2. `ChatApp` – корневой композабл

```kotlin
@Composable
fun ChatApp(
    mainViewModel: MainViewModel,
    repository: ChatRepository
) {
    val mainState by mainViewModel.state.collectAsState()
    val orientation = LocalConfiguration.current.orientation

    // Создаём или получаем ChannelsViewModel (живёт столько же, сколько Activity)
    val channelsViewModel: ChannelsViewModel = viewModel(
        factory = ChannelsViewModelFactory(repository, mainViewModel)
    )

    // Обработка системной кнопки "Назад"
    BackHandler {
        handleBackPress(mainViewModel, orientation)
    }

    if (!mainState.isAuthenticated) {
        // Экран входа
        LoginScreen(
            repository = repository,
            onLoginSuccess = { mainViewModel.checkAuth() }
        )
    } else {
        // Основной контент
        when (orientation) {
            Configuration.ORIENTATION_LANDSCAPE -> {
                LandscapeContent(
                    mainViewModel = mainViewModel,
                    channelsViewModel = channelsViewModel,
                    repository = repository
                )
            }
            else -> {
                PortraitContent(
                    mainViewModel = mainViewModel,
                    channelsViewModel = channelsViewModel,
                    repository = repository
                )
            }
        }
    }

    // Полноэкранная картинка поверх всего
    if (mainState.fullImagePath != null) {
        FullImageScreen(
            path = mainState.fullImagePath!!,
            onClose = { mainViewModel.hideImage() }
        )
    }
}
```

#### 3. `PortraitContent` – портретный режим

```kotlin
@Composable
fun PortraitContent(
    mainViewModel: MainViewModel,
    channelsViewModel: ChannelsViewModel,
    repository: ChatRepository
) {
    val selectedChannel by mainViewModel.state.collectAsState().selectedChannel

    if (selectedChannel == null) {
        ChannelsScreen(
            viewModel = channelsViewModel,
            onChannelClick = { channel ->
                mainViewModel.selectChannel(channel)
            },
            onLogout = { mainViewModel.logout() }
        )
    } else {
        // Создаём ChatViewModel с ключом = имени канала, чтобы сохранить состояние при повороте
        val chatViewModel: ChatViewModel = viewModel(
            key = selectedChannel,
            factory = ChatViewModelFactory(repository, mainViewModel, selectedChannel!!)
        )
        ChatScreen(
            viewModel = chatViewModel,
            channel = selectedChannel!!,
            onBack = { mainViewModel.selectChannel(null) },
            onImageClick = { imagePath -> mainViewModel.showImage(imagePath) }
        )
    }
}
```

#### 4. `LandscapeContent` – ландшафтный режим (двухпанельный)

```kotlin
@Composable
fun LandscapeContent(
    mainViewModel: MainViewModel,
    channelsViewModel: ChannelsViewModel,
    repository: ChatRepository
) {
    val selectedChannel by mainViewModel.state.collectAsState().selectedChannel

    Row(modifier = Modifier.fillMaxSize()) {
        // Левая панель – список чатов (всегда виден)
        Box(modifier = Modifier.weight(0.35f)) {
            ChannelsScreen(
                viewModel = channelsViewModel,
                onChannelClick = { channel ->
                    mainViewModel.selectChannel(channel)
                },
                onLogout = { mainViewModel.logout() }
            )
        }

        // Правая панель – детали чата или заглушка
        Box(modifier = Modifier.weight(0.65f)) {
            if (selectedChannel == null) {
                Box(
                    contentAlignment = Alignment.Center,
                    modifier = Modifier.fillMaxSize()
                ) {
                    Text("Выберите чат", style = MaterialTheme.typography.headlineSmall)
                }
            } else {
                val chatViewModel: ChatViewModel = viewModel(
                    key = selectedChannel,
                    factory = ChatViewModelFactory(repository, mainViewModel, selectedChannel!!)
                )
                ChatScreen(
                    viewModel = chatViewModel,
                    channel = selectedChannel!!,
                    onBack = { /* В ландшафте кнопка "назад" закрывает чат, но здесь её отдельно не показываем */ },
                    onImageClick = { imagePath -> mainViewModel.showImage(imagePath) },
                    showBackButton = false // на правой панели кнопка назад не нужна
                )
            }
        }
    }
}
```

### Обработка кнопки «Назад» (единая логика)

```kotlin
private fun handleBackPress(mainViewModel: MainViewModel, orientation: Int) {
    val state = mainViewModel.state.value
    // Если открыта картинка – закрыть её
    if (state.fullImagePath != null) {
        mainViewModel.hideImage()
        return
    }
    // Если в портрете открыт чат – закрыть чат (перейти к списку)
    if (orientation == Configuration.ORIENTATION_PORTRAIT && state.selectedChannel != null) {
        mainViewModel.selectChannel(null)
        return
    }
    // Если в ландшафте открыт чат – закрыть чат (очистить selectedChannel)
    if (orientation == Configuration.ORIENTATION_LANDSCAPE && state.selectedChannel != null) {
        mainViewModel.selectChannel(null)
        return
    }
    // Иначе – закрыть приложение
    // Для Activity: можно вызвать finish() или переместить в фон
    // В Compose нет прямого finish, можно отправить сигнал через колбэк в Activity
}
```

В `MainActivity` нужно передать колбэк `onExit` в `ChatApp`, чтобы можно было завершить Activity:
```kotlin
setContent {
    ChatApp(
        mainViewModel = mainViewModel,
        repository = repository,
        onExit = { finish() }
    )
}
```

Тогда в `handleBackPress` вместо комментария вызываем `onExit()`.

### Особенности `ChannelsScreen` и `ChatScreen` (краткое описание)

#### `ChannelsScreen`
- Отображает `LazyColumn` из списка каналов.
- Каждый элемент – `TextButton` с названием канала.
- `TopAppBar` с заголовком «Чаты» и кнопкой выхода (иконка выхода).
- При клике вызывает `onChannelClick`.
- При клике на выход – `onLogout`.

#### `ChatScreen`
- `LazyColumn` для сообщений (обратная прокрутка, пагинация при достижении верха).
- `TextField` для ввода текста и кнопка отправки (активна только если текст не пуст).
- `TopAppBar` с заголовком (имя канала) и кнопкой «Назад» (в портрете или если `showBackButton = true`).
- Каждое сообщение:
  - Текст → отображаем текст.
  - Картинка → показываем миниатюру из `/thumb/` (можно через `Coil` с преобразованием пути к `thumb`). При клике вызываем `onImageClick(fullPath)`, где полный путь из `MessageData.Image.link`.

### Создание ViewModel с помощью фабрик

Чтобы передать `ChatRepository` и `MainViewModel` в дочерние ViewModel, создадим простые фабрики:

```kotlin
class ChannelsViewModelFactory(
    private val repository: ChatRepository,
    private val mainViewModel: MainViewModel
) : ViewModelProvider.Factory {
    override fun <T : ViewModel> create(modelClass: Class<T>): T {
        return ChannelsViewModel(repository, mainViewModel) as T
    }
}

class ChatViewModelFactory(
    private val repository: ChatRepository,
    private val mainViewModel: MainViewModel,
    private val channel: String
) : ViewModelProvider.Factory {
    override fun <T : ViewModel> create(modelClass: Class<T>): T {
        return ChatViewModel(repository, mainViewModel, channel) as T
    }
}
```

Аналогично для `MainViewModelFactory`.

### Обработка пагинации в `ChatScreen`

- Загружаем первую партию в `LaunchedEffect(Unit)` или при изменении `channel`.
- При достижении конца списка (или при скролле вверх) вызываем `chatViewModel.loadMoreMessages()`.
- Используем `LazyColumn` с обратным порядком (`reverseLayout = true`), чтобы новые сообщения были внизу. Для загрузки старых сообщений при прокрутке вверх – добавляем `item` в начало списка для индикатора загрузки и триггера.

### Особенности работы с картинками

- Для миниатюр: используем `AsyncImage` из Coil, указывая URL `https://your-server.com/thumb/${imageLink}` (или формируем полный URL из `MessageData.Image.link`, заменяя `/img/` на `/thumb/`).
- По клику – вызываем `mainViewModel.showImage(fullLink)`, где `fullLink` – это значение `link` из `MessageData.Image`. В `FullImageScreen` загружаем полное изображение через `repository.getFullImage()` или прямо через Coil, но лучше через `ImageViewModel`, чтобы соответствовать ТЗ и не делать лишних запросов при повороте.

### Защита от перезагрузки при повороте

- `MainViewModel` остаётся в памяти.
- `ChannelsViewModel` один на всю Activity – загружает каналы один раз.
- `ChatViewModel` создаётся с ключом `selectedChannel`. При повороте `selectedChannel` сохраняется, поэтому тот же экземпляр `ChatViewModel` будет использован повторно, и сообщения не будут перезагружены.

### Соответствие ТЗ

- **Поворот не вызывает новые запросы** – да, т.к. ViewModel переиспользуются.
- **401 запускает вход заново** – через `mainViewModel.onUnauthorized()` вызывается `logout()` и очистка состояния.
- **Ошибки показываются в диалоге** – каждая ViewModel имеет поле `errorMessage`, в UI можно показывать `AlertDialog`.
- **Картинки из `/thumb/`** – при клике открываются на весь экран.
- **Пагинация 20 сообщений** – в `ChatViewModel` реализована.
- **Кнопка «Назад»** – работает по единой логике.
- **Хардкод строк** – нет, все строки вынесены в ресурсы.

### Итоговая схема связи

```
MainActivity
  └─ ChatApp (подписан на MainState)
       ├─ LoginScreen (если !isAuthenticated)
       └─ иначе:
            ├─ PortraitContent (содержит ChannelsScreen / ChatScreen)
            ├─ LandscapeContent (две панели)
            └─ FullImageScreen (если fullImagePath != null)
```

Это архитектура максимально проста, понятна и удовлетворяет всем требованиям задания.
