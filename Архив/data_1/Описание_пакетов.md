## ТЗ пакета `data.storage`

### Назначение
Обеспечить локальное сохранение и чтение аутентификационных данных (токен доступа, имя пользователя) между сессиями.  
Используется как источник данных для репозитория.

### Компоненты
- `AuthStorage` – основной класс для работы с хранилищем.

### Технические требования

#### Класс `AuthStorage`

**Пакет**: `com.ваш_домен.chat.data.storage` (или `ru.uni.chat.data.storage`).

**Конструктор**:
- Принимает `Context` (для доступа к `SharedPreferences`).  
- Рекомендуется использовать `context.applicationContext`, чтобы избежать утечек.

**Методы**:

1. `fun saveToken(token: String, userName: String)`  
   - Сохраняет две строки: `auth_token` и `user_name`.  
   - Не должен быть `suspend` (т.к. `SharedPreferences.apply()` не блокирует).

2. `fun getToken(): String?`  
   - Возвращает сохранённый токен или `null`.

3. `fun getUserName(): String?`  
   - Возвращает сохранённое имя пользователя или `null`.

4. `fun clear()`  
   - Удаляет оба сохранённых значения.

**Хранилище**:  
- Использовать `androidx.preference.PreferenceManager.getDefaultSharedPreferences` или отдельный файл `"chat_prefs"`.  
- Ключи: `"auth_token"`, `"user_name"`.

**Ограничения**:
- Никакой бизнес-логики – только чтение/запись.
- Никаких вызовов сетевых операций.
- Все поля внутри класса – `private val`.
- Никаких `var` вне методов.

**Пример реализации (каркас)**:
```kotlin
class AuthStorage(context: Context) {
    private val prefs = context.applicationContext.getSharedPreferences("chat_auth", Context.MODE_PRIVATE)

    fun saveToken(token: String, userName: String) {
        prefs.edit()
            .putString("auth_token", token)
            .putString("user_name", userName)
            .apply()
    }

    fun getToken(): String? = prefs.getString("auth_token", null)
    fun getUserName(): String? = prefs.getString("user_name", null)
    fun clear() = prefs.edit().clear().apply()
}
```

**Требования к качеству**:
- Нет warning: все строки-ключи вынесены в `private const val` внутри companion object.
- Не использует `@JvmStatic` без необходимости.
- Не содержит аннотаций, связанных с UI.

## ТЗ пакета `data.api`

### Назначение
Инкапсулировать все сетевые вызовы к серверу чата (REST API) с использованием Retrofit.  
Предоставляет чистый интерфейс `ChatApiService` для репозитория.

### Компоненты
- `ChatApiService` – интерфейс Retrofit (suspend-методы).
- DTO-классы (несколько `data class` и sealed class).
- `NetworkModule` (или отдельный файл) – создание экземпляра `ChatApiService` с настройками.

### Технические требования

#### 1. Настройки сети

- **Base URL**: `"https://faerytea.name/"`
- **Retrofit.Builder**:
  - `baseUrl(BASE_URL)`
  - `addConverterFactory(MoshiConverterFactory.create())` (или `kotlinx.serialization`)
  - `client(...)` – OkHttpClient с таймаутами и перехватчиками (опционально логгирование).
- **OkHttpClient**:
  - `connectTimeout(15, TimeUnit.SECONDS)`
  - `readTimeout(30, TimeUnit.SECONDS)`
  - Опционально: `addInterceptor(HttpLoggingInterceptor().setLevel(Level.BODY))` только для debug.

#### 2. Интерфейс `ChatApiService`

Все методы – `suspend`. Аннотации в соответствии с документацией сервера.

```kotlin
interface ChatApiService {
    @POST("login")
    @Headers("Content-Type: application/json")
    suspend fun login(@Body request: LoginRequest): LoginResponse

    @GET("channels")
    suspend fun getChannels(@Header("X-Auth-Token") token: String): List<String>

    @GET("channel/{channelName}")
    suspend fun getMessages(
        @Header("X-Auth-Token") token: String,
        @Path("channelName") channelName: String,
        @Query("limit") limit: Int,
        @Query("lastKnownId") lastKnownId: Int
    ): List<MessageDto>

    @POST("messages")
    @Headers("Content-Type: application/json")
    suspend fun sendTextMessage(
        @Header("X-Auth-Token") token: String,
        @Body message: MessageDto
    ): Response<Unit>

    @Multipart
    @POST("messages")
    suspend fun sendImageMessage(
        @Header("X-Auth-Token") token: String,
        @Part("msg") msg: MultipartBody.Part,
        @Part picture: MultipartBody.Part
    ): Response<Unit>

    @POST("logout")
    suspend fun logout(@Header("X-Auth-Token") token: String): Response<Unit>
}
```

#### 3. DTO-классы (пакет `data.api.dto`)

Все поля `val`, сериализуются стандартно.

**LoginRequest.kt**
```kotlin
data class LoginRequest(
    val name: String,
    val pwd: String
)
```

**LoginResponse.kt**
```kotlin
data class LoginResponse(
    val token: String
)
```

**MessageDto.kt**
```kotlin
data class MessageDto(
    val id: String? = null,
    val from: String,
    val to: String = "1@channel",
    val data: MessageData,
    val time: Long? = null
)
```

**MessageData.kt** – sealed class (или иерархия с полиморфизмом).
```kotlin
sealed class MessageData {
    data class TextMessage(val text: String) : MessageData()
    data class ImageMessage(val link: String? = null) : MessageData()
}
```
*Чтобы сериализатор правильно работал с `sealed class`, нужна соответствующая поддержка (Moshi с `PolymorphicJsonAdapter` или kotlinx.serialization с аннотацией `@SerialName`).*

#### 4. Фабрика `NetworkModule`

**Не обязан быть полноценным DI**, достаточно `object` с функцией `provideApiService(): ChatApiService`.

```kotlin
object NetworkModule {
    private const val BASE_URL = "https://faerytea.name/"

    fun createApiService(): ChatApiService {
        val okHttpClient = OkHttpClient.Builder()
            .connectTimeout(15, TimeUnit.SECONDS)
            .readTimeout(30, TimeUnit.SECONDS)
            .addInterceptor(HttpLoggingInterceptor().apply {
                level = if (BuildConfig.DEBUG) HttpLoggingInterceptor.Level.BODY else Level.NONE
            })
            .build()

        val retrofit = Retrofit.Builder()
            .baseUrl(BASE_URL)
            .client(okHttpClient)
            .addConverterFactory(MoshiConverterFactory.create())
            .build()

        return retrofit.create(ChatApiService::class.java)
    }
}
```

**Примечание по `BuildConfig`** – может отсутствовать в модуле `data`, если не подключен плагин `com.android.library`. В учебных целях можно проверять `BuildConfig.DEBUG` через `BuildConfig` из модуля `app` – но лучше передавать флаг извне. Для упрощения допустим всегда Level.NONE.

#### 5. Требования к обработке ошибок

- Не перехватывать исключения внутри `ChatApiService`. Ошибки HTTP (401, 404, 500) пробрасываются как `HttpException` из Retrofit.
- Не модифицировать ответы (например, не оборачивать в `Result`). Этим занимается репозиторий.

#### 6. Качество кода

- Нет `var` – все DTO immutable.
- Все публичные типы имеют документацию (краткую).
- Нет захардкоженных строк в методах – только константы.
- Нет варнингов (например, неиспользуемые параметры помечать `@Suppress("UNUSED_PARAMETER")` если необходимо).

## ТЗ пакета `data.repository`

### Назначение
Предоставить высокоуровневый репозиторий для работы с чатом, скрывая детали сети и хранения.  
Координирует вызовы `ChatApiService` и `AuthStorage`, обрабатывает ошибки (в частности, превращает `401` в `UnauthorizedException`).

### Компоненты
- `ChatRepository` – основной класс.
- `UnauthorizedException` – кастомное исключение.

### Технические требования

#### 1. Класс `ChatRepository`

**Пакет**: `com.ваш_домен.chat.data.repository`

**Конструктор**:
```kotlin
class ChatRepository(
    private val api: ChatApiService,
    private val authStorage: AuthStorage
)
```

**Все публичные методы – `suspend`**, кроме `getImageUrl`.

**Возвращаемые значения**: `Result<T>` (стандартный `kotlin.Result`).  
В случае ошибки возвращается `Result.failure(exception)`. При успехе – `Result.success(data)`.  
Если требуется вернуть `Unit` (например, при логине) – `Result<Unit>`.

#### 2. Методы репозитория

##### `suspend fun login(name: String, password: String): Result<Unit>`
- Вызывает `api.login(LoginRequest(name, pwd))`.
- При успехе сохраняет токен и имя пользователя: `authStorage.saveToken(response.token, name)`.
- Возвращает `Result.success(Unit)`.
- При любом исключении: `Result.failure(exception)`.

##### `suspend fun logout(): Result<Unit>`
- Если токен есть (не `null`), пытается вызвать `api.logout(token)`. Ошибки при этом игнорируются (логировать опционально).
- В любом случае вызывает `authStorage.clear()`.
- Возвращает `Result.success(Unit)`.

##### `suspend fun getChannels(): Result<List<String>>`
- Получает токен: `val token = authStorage.getToken()`. Если `null` → `UnauthorizedException`.
- Вызывает `api.getChannels(token)`.
- При `HttpException` с кодом `401` → `Result.failure(UnauthorizedException())`.
- При других ошибках → `Result.failure(exception)`.
- При успехе → `Result.success(channels)`.

##### `suspend fun getMessages(channelName: String, limit: Int = 20, lastKnownId: Int = 0): Result<List<MessageDto>>`
- Аналогично: проверка токена, вызов `api.getMessages(token, channelName, limit, lastKnownId)`, обработка 401.

##### `suspend fun sendTextMessage(channelName: String, text: String): Result<Unit>`
- Проверка токена.
- Получает `userName = authStorage.getUserName()`. Если `null` → `UnauthorizedException`.
- Создаёт `MessageDto(from = userName, to = channelName, data = MessageData.TextMessage(text))`.
- Вызывает `api.sendTextMessage(token, message)`. Ответ `Response<Unit>` – проверяет, что `response.isSuccessful`. Если нет – выбрасывает `HttpException`.
- Обработка 401 по аналогии.
- При успехе → `Result.success(Unit)`.

##### `suspend fun sendImageMessage(channelName: String, imageFile: File, caption: String? = null): Result<Unit>`
- Проверка токена и имени пользователя.
- Создаёт `MessageDto`:
  - Если `caption` не null и не пуст → `data = MessageData.TextMessage(caption)` (согласно заданию: при отправке картинки можно добавить текстовое описание).
  - Иначе → `data = MessageData.ImageMessage()` (без link).
- Сериализует этот DTO в JSON-строку (используя тот же конвертер, что и Retrofit). Например, через `Moshi.Builder().build().adapter(MessageDto::class.java).toJson(message)`.
- Создаёт `MultipartBody.Part` для `msg`:
  ```kotlin
  val jsonPart = MultipartBody.Part.createFormData(
      "msg", null,
      jsonString.toRequestBody("application/json".toMediaType())
  )
  ```
- Создаёт `MultipartBody.Part` для `picture`:
  ```kotlin
  val imagePart = MultipartBody.Part.createFormData(
      "picture", imageFile.name,
      imageFile.asRequestBody("image/*".toMediaType())
  )
  ```
- Вызывает `api.sendImageMessage(token, jsonPart, imagePart)`.
- Проверяет успешность ответа.
- Обрабатывает 401.
- Возвращает `Result<Unit>`.

##### `fun getImageUrl(path: String, thumb: Boolean): String`
- **Не suspend**.
- `thumb == true` → `"https://faerytea.name/thumb/$path"`
- `thumb == false` → `"https://faerytea.name/img/$path"`
- Не выполняет сетевых вызовов.

#### 3. Исключение `UnauthorizedException`

```kotlin
class UnauthorizedException : Exception("Authentication token is missing or expired")
```

#### 4. Важные требования

- **Никогда** не очищать токен внутри репозитория при получении 401. Это ответственность ViewModel.
- Не ловить `HttpException` кроме как для обработки 401; в остальных случаях пробрасывать дальше.
- Для сериализации JSON внутри `sendImageMessage` – можно использовать экземпляр `Moshi`, созданный через `NetworkModule`, чтобы быть совместимым с Retrofit.
- Все строки (URL для картинок) должны быть константами.
- Избегать `var`, мутабельных коллекций, состояний.

#### 5. Пример использования (не входит в реализацию пакета)

```kotlin
val result = repository.getChannels()
result.onSuccess { list -> /* обновить UI */ }
     .onFailure { e ->
         if (e is UnauthorizedException) // навигация на логин
         else // показать ошибку
     }
```
