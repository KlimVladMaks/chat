## Архитектура модуля `viewmodel`

В соответствии с ТЗ и с учётом уже готового модуля `data` (ChatRepository), предлагается следующая организация `viewmodel`:

- **`MainViewModel`** – общее состояние приложения (аутентификация, выбранный канал, отображаемое полноэкранное изображение). Живёт в области видимости `Activity` и сохраняется при повороте.
- **`LoginViewModel`** – управляет экраном входа.
- **`ChannelsViewModel`** – управляет списком каналов.
- **`ChatViewModel`** – управляет сообщениями выбранного канала (пагинация, отправка).
- **`ImageViewModel`** – загружает полноразмерное изображение для экрана картинки (опционально, но упрощает тестирование).

Все ViewModel используют `StateFlow` для публикации неизменяемого состояния и обрабатывают `UnauthorizedException` через централизованное событие в `MainViewModel`. При повороте сетевые вызовы не повторяются, так как данные уже закешированы во `Flow` ViewModel.

---

## Детальное описание ViewModel

### 1. `MainViewModel`

**Состояние** (immutable data class):
```kotlin
data class MainState(
    val isAuthenticated: Boolean = false,
    val selectedChannel: String? = null,
    val fullImagePath: String? = null,
    val username: String? = null
)
```

**События / методы**:
- `checkAuth()` – при старте проверяет наличие токена в `AuthStorage`, обновляет `isAuthenticated`.
- `selectChannel(channel: String)` – обновляет `selectedChannel`.
- `showImage(path: String)` – устанавливает `fullImagePath`.
- `hideImage()` – сбрасывает `fullImagePath`.
- `logout()` – вызывает `repository.logout()`, очищает состояние, обновляет `isAuthenticated = false`.
- `onUnauthorized()` – обработчик `UnauthorizedException` из дочерних ViewModel (вызовет `logout`).
- `sendAuthError(errorMessage: String)` – для показа диалога ошибки (например, при логине).

Все изменения состояния выполняются через `_state.update { ... }`.

**Взаимодействие с UI**:
- В портретном режиме навигация происходит через `NavController`, но состояние `selectedChannel` и `fullImagePath` хранится в `MainViewModel`, чтобы при повороте не терять открытый чат или картинку.
- При изменении `isAuthenticated` UI переключает граф навигации (если `false` → экран входа, если `true` → экран чатов/сообщений).
- При изменении `fullImagePath` UI показывает полноэкранное окно с картинкой поверх всего.

### 2. `LoginViewModel`

**Состояние**:
```kotlin
data class LoginState(
    val username: String = "",
    val password: String = "",
    val isLoading: Boolean = false,
    val errorMessage: String? = null
)
```

**Методы**:
- `updateUsername(String)`, `updatePassword(String)`
- `login(repository, mainViewModel)` – вызывает `repository.login`, при успехе – `mainViewModel.checkAuth()` (или прямой вызов `mainViewModel.setAuthenticated(true)`), при ошибке выставляет `errorMessage`.  
  При `UnauthorizedException` – показывает «Неверный логин/пароль».

**Особенности**:
- Если `mainViewModel.isAuthenticated == true`, экран входа вообще не показывается (проверка на уровне навигации).

---

### 3. `ChannelsViewModel`

**Состояние**:
```kotlin
data class ChannelsState(
    val channels: List<String> = emptyList(),
    val isLoading: Boolean = false,
    val error: String? = null
)
```

**Методы**:
- `loadChannels(repository, mainViewModel)` – загружает каналы при создании (если `channels` пуст), кеширует результат. При `UnauthorizedException` вызывает `mainViewModel.onUnauthorized()`.
- `onChannelClick(channel: String, mainViewModel, navController, isLandscape: Boolean)`  
  В ландшафте: `mainViewModel.selectChannel(channel)`.  
  В портрете: `mainViewModel.selectChannel(channel)` + `navController.navigate("chat/${channel}")`.

**Сохранение при повороте**:
- `channels` сохраняется во `StateFlow`, повторная загрузка не происходит.

### 4. `ChatViewModel`

Конструктор принимает `channel: String?` (через `SavedStateHandle`), чтобы реагировать на изменение канала.

**Состояние**:
```kotlin
data class ChatState(
    val channelName: String? = null,
    val messages: List<Message> = emptyList(),
    val isLoadingInitial: Boolean = false,
    val isLoadingMore: Boolean = false,
    val hasMore: Boolean = true,
    val sendText: String = "",
    val isSending: Boolean = false,
    val error: String? = null
)
```

**Методы**:
- `loadInitialMessages(repository, mainViewModel)` – загружает первые 20 сообщений, сбрасывает `messages`. При ошибке 401 вызывает `mainViewModel.onUnauthorized()`.
- `loadMoreMessages(repository, mainViewModel)` – загружает следующие 20, используя `lastKnownId = messages.lastOrNull()?.id`.
- `updateSendText(String)`
- `sendMessage(repository, mainViewModel)` – отправляет текст, после успеха сбрасывает `sendText` и повторно вызывает `loadInitialMessages()` (для получения обновлённого списка с сервера).

**Особенности**:
- При изменении `channelName` (например, пользователь выбрал другой канал) автоматически перезагружаем сообщения.
- `ChatViewModel` не сохраняется в `NavGraph` при портрете, но в двухпанельном ландшафте он может пересоздаваться при смене канала. Однако данные не будут теряться, так как каждый экземпляр `ChatViewModel` относится к конкретному каналу. Чтобы избежать повторной загрузки при повороте, в ландшафте `ChatViewModel` должен жить в рамках `MainActivity`, а не в `NavHost`. Альтернатива – в ландшафте вообще не использовать `NavHost`, а иметь два фиксированных фрагмента, и `ChatViewModel` получать через `viewModel()` с ключом по имени канала.

**Упрощённое решение для ландшафта**:
В правой панели используем `ChatScreen` с `ChatViewModel`, созданным через `keyed viewModel`:
```kotlin
val chatViewModel: ChatViewModel = viewModel(
    key = selectedChannel ?: "none",
    factory = chatViewModelFactory(selectedChannel)
)
```
Такой подход гарантирует, что при повороте ViewModel для того же канала сохранится и не будет перезагружать сообщения.

### 5. `ImageViewModel`

**Состояние**:
```kotlin
data class ImageState(
    val isLoading: Boolean = true,
    val bitmap: Bitmap? = null,
    val error: String? = null
)
```

**Методы**:
- `loadImage(repository, path, mainViewModel)` – загружает полное изображение через `repository.getFullImage()` и превращает `ResponseBody` в `Bitmap`. При 401 вызывает `mainViewModel.onUnauthorized()`.

**Примечание**:
В Compose можно обойтись без отдельной ViewModel, загружая картинку через `remember` + `LaunchedEffect`, но ViewModel облегчает тестирование и повторное использование при повороте.

## Обработка навигации и кнопки «Назад»

- **Портрет**:
  - Экран чатов → нажатие на канал → `navController.navigate("chat/{channel}")`.
  - Экран сообщений → кнопка «Назад» (системная или жест) → `navController.popBackStack()`.
  - Экран картинки → кнопка «Назад» → `mainViewModel.hideImage()`.
  - На экране чатов кнопка «Назад» → закрыть приложение.

- **Ландшафт**:
  - Слева всегда список чатов. Кнопка «Назад» закрывает открытый чат: если `selectedChannel != null` → `mainViewModel.selectChannel(null)`, иначе закрыть приложение.
  - Экран картинки (поверх всего) → кнопка «Назад» закрывает картинку.

Реализуется через `BackHandler` в каждом composable, проверяя текущее состояние ориентации и `selectedChannel` / `fullImagePath`.

---

## Взаимодействие с `ChatRepository` и обработка ошибок

1. Все ViewModel получают экземпляр `ChatRepository` через внедрение зависимостей (например, ручное создание в `Application` или через DI-библиотеку типа Dagger/Hilt, но для простоты – передача в конструктор при создании ViewModel).
2. Вызовы методов репозитория обёрнуты в `try-catch`:
   - При `UnauthorizedException` вызывается `mainViewModel.onUnauthorized()`.
   - При других исключениях – выставляется `error` в состоянии для отображения диалога.
3. `MainViewModel.onUnauthorized()` вызывает `logout()` и обновляет `isAuthenticated = false`, что переключает навигацию на экран входа.

## Пример кода (ключевые фрагменты)

### `MainViewModel`
```kotlin
class MainViewModel(private val repository: ChatRepository) : ViewModel() {
    private val _state = MutableStateFlow(MainState())
    val state: StateFlow<MainState> = _state.asStateFlow()

    fun checkAuth() {
        val token = repository.authStorage.getToken()
        val username = repository.authStorage.getUsername()
        _state.update { it.copy(isAuthenticated = token != null, username = username) }
    }

    fun selectChannel(channel: String?) {
        _state.update { it.copy(selectedChannel = channel) }
    }

    fun showImage(path: String) {
        _state.update { it.copy(fullImagePath = path) }
    }

    fun hideImage() {
        _state.update { it.copy(fullImagePath = null) }
    }

    fun logout() {
        viewModelScope.launch {
            repository.logout()
            _state.update { MainState() }
        }
    }

    fun onUnauthorized() {
        viewModelScope.launch {
            repository.logout()
            _state.update { MainState() }
        }
    }
}
```

### `LoginViewModel`
```kotlin
class LoginViewModel(private val repository: ChatRepository) : ViewModel() {
    private val _state = MutableStateFlow(LoginState())
    val state: StateFlow<LoginState> = _state.asStateFlow()

    fun updateUsername(value: String) {
        _state.update { it.copy(username = value, errorMessage = null) }
    }
    fun updatePassword(value: String) {
        _state.update { it.copy(password = value, errorMessage = null) }
    }

    fun login(mainViewModel: MainViewModel, onSuccess: () -> Unit) {
        viewModelScope.launch {
            _state.update { it.copy(isLoading = true, errorMessage = null) }
            try {
                val token = repository.login(_state.value.username, _state.value.password)
                mainViewModel.checkAuth() // обновит isAuthenticated
                onSuccess()
            } catch (e: UnauthorizedException) {
                _state.update { it.copy(errorMessage = "Неверный логин/пароль") }
            } catch (e: Exception) {
                _state.update { it.copy(errorMessage = e.message ?: "Ошибка входа") }
            } finally {
                _state.update { it.copy(isLoading = false) }
            }
        }
    }
}
```

### `ChannelsViewModel`
```kotlin
class ChannelsViewModel(private val repository: ChatRepository) : ViewModel() {
    private val _state = MutableStateFlow(ChannelsState())
    val state: StateFlow<ChannelsState> = _state.asStateFlow()

    fun loadChannels(mainViewModel: MainViewModel) {
        if (_state.value.channels.isNotEmpty()) return
        viewModelScope.launch {
            _state.update { it.copy(isLoading = true, error = null) }
            try {
                val channels = repository.getChannels()
                _state.update { it.copy(channels = channels) }
            } catch (e: UnauthorizedException) {
                mainViewModel.onUnauthorized()
            } catch (e: Exception) {
                _state.update { it.copy(error = e.message) }
            } finally {
                _state.update { it.copy(isLoading = false) }
            }
        }
    }

    fun onChannelClick(channel: String, mainViewModel: MainViewModel, navController: NavController, isLandscape: Boolean) {
        mainViewModel.selectChannel(channel)
        if (!isLandscape) {
            navController.navigate("chat/$channel")
        }
    }
}
```

### `ChatViewModel` (упрощённо)
```kotlin
class ChatViewModel(
    private val repository: ChatRepository,
    private val savedStateHandle: SavedStateHandle
) : ViewModel() {
    private val _state = MutableStateFlow(ChatState())
    val state: StateFlow<ChatState> = _state.asStateFlow()

    private val channel: String?
        get() = savedStateHandle["channel"]

    init {
        viewModelScope.launch {
            savedStateHandle.getStateFlow("channel", null).collect { _ ->
                loadInitialMessages()
            }
        }
    }

    fun loadInitialMessages(mainViewModel: MainViewModel) {
        val currentChannel = channel ?: return
        viewModelScope.launch {
            _state.update { it.copy(isLoadingInitial = true, error = null) }
            try {
                val msgs = repository.getMessages(currentChannel, limit = 20)
                _state.update {
                    it.copy(
                        messages = msgs,
                        isLoadingInitial = false,
                        hasMore = msgs.size == 20
                    )
                }
            } catch (e: UnauthorizedException) {
                mainViewModel.onUnauthorized()
            } catch (e: Exception) {
                _state.update { it.copy(error = e.message, isLoadingInitial = false) }
            }
        }
    }

    fun sendMessage(text: String, mainViewModel: MainViewModel) {
        if (text.isBlank()) return
        val currentChannel = channel ?: return
        viewModelScope.launch {
            _state.update { it.copy(isSending = true) }
            try {
                repository.sendTextMessage(currentChannel, text)
                _state.update { it.copy(sendText = "") }
                loadInitialMessages(mainViewModel) // обновить список
            } catch (e: UnauthorizedException) {
                mainViewModel.onUnauthorized()
            } catch (e: Exception) {
                _state.update { it.copy(error = e.message) }
            } finally {
                _state.update { it.copy(isSending = false) }
            }
        }
    }

    // loadMoreMessages реализуется аналогично
}
```

## Итог

Предложенная архитектура:

- **Модульность** – каждый ViewModel отвечает за свой экран.
- **Сохранение состояния при повороте** – данные живут во `Flow` ViewModel, сетевые вызовы делаются только при необходимости.
- **Обработка 401** – централизованно через `MainViewModel.onUnauthorized()`.
- **Навигация и кнопка «Назад»** – управляется через `BackHandler` и состояние `MainViewModel`.
- **Соответствие ТЗ** – учтены все требования (пагинация, поворот, картинки через `/thumb/`, диалоги ошибок и т.д.).

Такой подход является оптимальным, простым и понятным для учебного проекта на Jetpack Compose.
