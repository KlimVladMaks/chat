# Как тут ходить

Сервер лежит по адресу https://faerytea.name/

## Регистрация

-   Web:  /adduser.html. Написать имя пользователя, получить пароль, запомнить.
-   API: POST на  `/addusr`, в  **теле**  запроса  `name=<my name>`.  `<my name>`  заменить на имя пользователя. Ответом придёт plain text вида  `password: '<password>'`.

## Логин

POST на  `/login`  JSON следующего вида:

```json
{ "name": "<имя пользователя>", "pwd": "<пароль>" }
```

В ответ придёт токен. Токен кладётся в заголовок  `X-Auth-Token`. Токен живет до рестарта сервера.

Токен просто приходит в формате "text/plain; charset=UTF-8". Например, вот так:
```
H+/OOSZ7Qr2S05A2AAABngNL260=
```

Куда нужен токен:

-   GET  `/inbox/<username>`
-   POST  `/messages`
-   POST  `/logout`

## Что тут бывает

### Сообщение

```json
{
  "id": "123", // игнорируется в запросах, обязательно будет в ответе
  "from": "my name", // обязательное, "от кого", должно совпадать с именем пользователя
  "to": "1@channel", // опциональное в запросах, по умолчанию "1@channel"
  "data": {
    "Text": {
      "text": "my message" // обязательно
    },
    // ИЛИ (исключающее)
    "Image": {
      "link": "path/to/pic.jpg" // опционально для запросов
    }
  },
  "time": "1297490" // UNIX time, опционально для запросов
}
```

### Списки

```json
["elem1", "elem2", "elem3"]
```

-   Список каналов
-   Список пользователей

## Куда и как ходить

### Получение сообщений

-   GET  `/1ch`  
    Список сообщений из канала  `1@channel`, то же самое, что и  `/channel/1@channel`
-   GET  `/channel/<имя канала>`  
    Список сообщений из указанного канала

Параметры запросов (для сообщений):

-   `limit`  
    Сколько сообщений отдавать, по умолчанию 20
-   `lastKnownId`  
    Начиная с какого сообщения отдавать сообщения, по умолчанию 0
-   `reverse`  
    Возвращает сообщения в обратном порядке (с  `id`  _меньше_, чем  `lastKnownId`), по умолчанию  `false`

### Получение информации

-   GET  `/channels`  
    Список каналов в виде JSON массива строк
-   GET  `/users`  
    Список  _зарегистрированных_  пользователей в виде JSON массива строк
-   GET  `/img/<path>`  
    Картиночка по указанному пути в хорошем разрешении
-   GET  `/thumb/<path>`  
    Картиночка по указанному пути в шакальном разрешении
-   GET  `/typing/<channel>`  
    Кто тут печатает

### Отправка информации

-   POST  `/1ch`  (или  `/messages`) с сообщением в body и  `Content-Type=application/json`
    -   `200 OK`  и  `id`  поста
    -   Ошибку, смотри коды состояния HTTP
-   POST  `/logout`  
    Забывает токен, всегда успешно

## А как сделать чат?

Просто напишите в него!

Таким же образом можно сделать канал, написав в  `<что угодно>@channel`

package io.github.klimvladmaks.chat.data.api

    @GET("/thumb/{path}")

package io.github.klimvladmaks.chat.data.api

import com.jakewharton.retrofit2.converter.kotlinx.serialization.asConverterFactory
import io.github.klimvladmaks.chat.data.storage.AuthStorage
import kotlinx.serialization.json.Json
import okhttp3.Authenticator
import okhttp3.Interceptor
import okhttp3.MediaType.Companion.toMediaType
import okhttp3.OkHttpClient
import okhttp3.Request
import okhttp3.Response
import okhttp3.Route
import retrofit2.Retrofit
import retrofit2.converter.scalars.ScalarsConverterFactory
import java.util.concurrent.TimeUnit

object RetrofitClient {
    private const val BASE_URL = "https://faerytea.name/"

    fun createApiService(authStorage: AuthStorage): ApiService {
        val client = OkHttpClient.Builder()
            .addInterceptor(AuthInterceptor(authStorage))
            .connectTimeout(30, TimeUnit.SECONDS)
            .readTimeout(30, TimeUnit.SECONDS)
            .build()

        val contentType = "application/json".toMediaType()
        val json = Json { ignoreUnknownKeys = true }
        val retrofit = Retrofit.Builder()
            .baseUrl(BASE_URL)
            .addConverterFactory(ScalarsConverterFactory.create())
            .addConverterFactory(json.asConverterFactory(contentType))
            .client(client)
            .build()

        return retrofit.create(ApiService::class.java)
    }
}

class AuthInterceptor(private val authStorage: AuthStorage) : Interceptor {
    override fun intercept(chain: Interceptor.Chain): Response {
        val originalRequest = chain.request()
        val token = authStorage.getToken()
        val request = if (token != null) {
            originalRequest.newBuilder()
                .header("X-Auth-Token", token)
                .build()
        }
        else {
            originalRequest
        }
        return chain.proceed(request)
    }
}

package io.github.klimvladmaks.chat.data.storage

import android.content.Context
import android.content.SharedPreferences
import androidx.core.content.edit

class AuthStorage(context: Context) {
    private val prefs: SharedPreferences = context.applicationContext.getSharedPreferences(
        PREFS_NAME,
        Context.MODE_PRIVATE
    )

    fun saveTokenAndUsername(token: String, username: String) {
        prefs.edit {
            putString(KEY_TOKEN, token)
            putString(KEY_USERNAME, username)
        }
    }

    fun hasToken(): Boolean = getToken() != null

    fun hasUsername(): Boolean = getUsername() != null

    fun getToken(): String? = prefs.getString(KEY_TOKEN, null)

    fun getUsername(): String? = prefs.getString(KEY_USERNAME, null)

    fun clear() {
        prefs.edit { clear() }
    }

    private companion object {
        private const val PREFS_NAME = "chat_auth"
        private const val KEY_TOKEN = "auth_token"
        private const val KEY_USERNAME = "user_name"
    }
}
