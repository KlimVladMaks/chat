# Чат

## Экраны

В приложении ожидаются следующие экраны:

### Экран входа

- Поле для ввода логина
- Поле для ввода пароля
- Кнопка входа
- Адекватная реакция на ошибки (не '401: Unauthorized' в логи, а 'Неверный логин / пароль' в диалоговом окне)
- Пропускается, если логин и пароль уже вводились

### Экран чатов (портрет)

- Список чатов
  - Каналы (см `/channels`)
  - Личные сообщения (будут появляться в `/inbox`) пока реализовывать не нужно
  - Нажатие на чат открывает его
- Кнопка выхода (ведёт на экран входа)

### Экран сообщений (портрет)

- Подгружает (и показывает) сообщения из чата
  - Текстовые как есть
  - Картинки из `/thumb/`, по клику открывает на весь экран
  - Пагинация: загружать не более 20 сообщений за раз
- Поле для ввода текста (в рамках данного задания нужно реализовать лишь функционал отправки текста, функционал отправки изображений реализовывать НЕ НУЖНО)
- Кнопка отправки сообщения (отправляет непустое сообщение)
- Кнопка перехода на экран чатов

### Экран картинки

- Картинка из `/img/` на весь экран
- Адекватный способ её закрыть (переходя на экран чата)

### Экран чатов и сообщений (ландшафт)

- Слева содержимое экрана чатов
- Справа надпись "выберите чат" или содержимое экрана сообщений
- Открытый чат должен быть как-то обозначен в списке

## Общее

- При повороте должен сохраняться открытый чат (даже если во время поворота открыта картинка)
- Кнопка (при трёхкнопочном интерфейсе) и жест (при жестовом) "назад" должны вести себя ожидаемо
  - Не надо разлогинивать из списка чатов
  - На ландшафте кнопка должна закрывать чат, если есть
  - На портрете переходить с экрана сообщений на экран чатов
  - На картинке закрывать картинку
  - В остальных случаях закрывать приложение
- Поворот не должен вызывать поход в сеть
- 401: Unauthorized всегда запускает процесс входа заново
- Ожидается использование каких-нибудь адекватных библиотек для работы с сетью
  - Volley и голый java.net к ним не относятся, системный Apache HTTP Client тем более (особенно учитывая, что он удалён).

## Нежелательно

- Любой warning студии (жёлтое подчёркивание)
- Использование мутабельных (или вероятно мутабельных) объектов для состояния

## Всё плохо

- Пакет приложения начинается с `com.example`
- Не собирается
- Падает
- Заблокирован поворот в манифесте
- Хардкод строк
  - Alt+Enter → Extract string resource…

# Дополнительная информация

* При разработке используется подход Jetpack Compose.
* Это учебный проект, так что он должен быть максимально простым и понятным, но при этом соответствовать описанному выше заданию.
* В рамках данного задания нужно реализовать лишь функционал отправки текст, функционал для отправки изображений реализовывать не нужно.
